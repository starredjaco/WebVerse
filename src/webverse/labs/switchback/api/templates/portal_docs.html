How to use the Referral Details API
This is the partner-safe endpoint for validating referral metadata in staging.
Endpoint
Method
GET
Path
/v1/referrals/details
Query
code=SWITCH-2026-DEMO
Example request
curl -s "http://api.{{ lab_domain }}/v1/referrals/details?code=SWITCH-2026-DEMO"
Example response (success)
{
  "ok": true,
  "referral": {
    "code": "SWITCH-2026-DEMO",
    "referrer_email": "promo@marketing.switchback.local",
    "points": 42,
    "created_at": "2026-02-28T00:00:00Z"
  }
}
Example response (not found)
{
  "ok": false,
  "status": "not_found"
}
Tip: For a human-friendly lookup, use /portal/verify.
Legacy / old endpoints
These endpoints exist for backward compatibility with older integrations. They intentionally return minimal information and may be unsuitable for partner support.
GET /v1/referrals/lookup
Legacy
Constant-envelope response by design. This endpoint is used by older attribution tooling and does not expose referral details in the response body.
Example request
curl -s "http://api.{{ lab_domain }}/v1/referrals/lookup?code=SWITCH-2026-DEMO"
Example response
{
  "ok": true,
  "status": "received"
}
Migration guidance
Partners should migrate to /v1/referrals/details for staging validation and support workflows. Use the portal verification tool for human review.
